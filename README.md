# Travel Places Demo

**Travel Places Demo** — демонстраційний Android-застосунок для лабораторної роботи з теми **"Навігація в Jetpack Compose"**.

Проєкт показує, як реалізувати застосунок із трьома екранами, ViewModel, списком даних у пам'яті та навігацією з передачею аргумента `id`.

## Предметна область

Застосунок дозволяє вести список місць для майбутніх мандрівок.

Користувач може:

- переглядати список місць;
- додавати нове місце;
- переглядати деталі місця;
- редагувати існуюче місце;
- видаляти місце;
- змінювати рівень зацікавленості через Slider;
- позначати місце як відвідане.

## Екрани

### 1. PlacesListScreen

Головний екран зі списком місць.

Функціональність:

- показує назву місця, країну, категорію та рівень зацікавленості;
- кнопка `+` відкриває екран додавання;
- натискання на елемент списку відкриває екран деталей з передачею `placeId`.

### 2. AddEditPlaceScreen

Екран додавання або редагування місця.

Функціональність:

- введення назви;
- введення країни / регіону;
- введення опису;
- вибір категорії;
- збереження або скасування.

### 3. PlaceDetailsScreen

Екран деталей місця.

Функціональність:

- отримує `placeId` як аргумент;
- показує повну інформацію про місце;
- дозволяє змінити рівень зацікавленості;
- дозволяє позначити місце як відвідане;
- містить кнопки редагування, видалення та повернення назад.

## Основна навчальна ідея

Між екранами передається не весь об'єкт, а тільки його `id`:

```kotlin
navController.navigate(AppRoutes.placeDetails(place.id))
```

На цільовому екрані `id` отримується з аргументів маршруту:

```kotlin
val placeId = backStackEntry.arguments?.getInt("placeId")
```

Після цього ViewModel знаходить потрібний об'єкт у списку.

## Важливо

Дані зберігаються тільки у пам'яті ViewModel. Після повного перезапуску застосунку зміни не зберігаються. Це зроблено спеціально, щоб у цій лабораторній роботі зосередитися на навігації, передачі аргументів і роботі зі станом.
